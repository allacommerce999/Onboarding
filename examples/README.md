# Synthetic onboarding access examples

Эта папка содержит синтетические документы для разработки onboarding-агента.

Все ФИО, организации, ресурсы, учетные записи, IP-адреса, владельцы и номера согласований искусственные. Примеры сохраняют только принципы структуры:

- заявка содержит адресата, отправителя, дату, основание, сотрудника, ресурсы, роли доступа и согласование;
- учетная запись пользователя строится как `surname + initials`;
- сервисная учетная запись строится как `svc_ + system + purpose`;
- серверные ресурсы именуются по принципу `тип-система-номер`;
- чувствительные доступы требуют отдельного согласования;
- пароли, токены и ключи не указываются.

## Состав

| Файл | Назначение |
|------|------------|
| `synthetic_access_request_ad_domain.md` | Пример заявки на домен, почту, мессенджер и базовые сетевые ресурсы |
| `synthetic_access_request_business_systems.md` | Пример заявки на бизнес-систему и BI |
| `synthetic_access_request_servers_ssh_rdp.md` | Пример заявки на серверы с SSH/RDP |
| `synthetic_access_request_service_account.md` | Пример заявки на сервисную учетную запись |
| `synthetic_access_request_bundle_enterprise_systems.md` | Пример пакета заявок для рабочей группы 1С, Парус, БД и Cognos |
| `synthetic_cover_letter_access.md` | Пример сопроводительного письма к заявке |
| `synthetic_domain_accounts_register.md` | Пример реестра доменных учетных записей |
| `synthetic_onboarding_package_backend.md` | Пример полного onboarding-пакета для нового сотрудника |
| `synthetic_onboarding_package_enterprise_systems.md` | Пример полного onboarding-пакета для инженера корпоративных систем |

Персональные synthetic drafts по тестовым сотрудникам лежат в `../employees/`.

## Как использовать

1. Использовать файлы из `templates/` как базовую форму.
2. Использовать файлы из `examples/` как few-shot примеры для агента.
3. Не переносить реальные данные из `doc/` в генерируемые результаты.
4. Все неизвестные значения помечать как `требует уточнения`.
