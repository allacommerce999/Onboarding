# Onboarding templates

Эта папка содержит public-safe шаблоны onboarding-агента. Все реальные данные, локальные инструкции, маршруты доступа, ФИО, телефоны, email, IP, кабинеты, пропуска и номера автомобилей должны оставаться вне public-kit.

## Структура

| Папка | Что внутри |
|-------|------------|
| `core/` | базовая форма входных данных, шаблон onboarding-пакета, prompt template и quality checklist |
| `access/` | матрицы доступов и шаблоны заявок на ИТ-ресурсы |
| `instructions/` | шаблоны инструктажей и матрица обязательных процедур |
| `checklists/` | простые чек-листы по ПВТР, ОТ, ПБ, ИБ, пропускам, парковке, высоте и электроустановкам |
| `knowledge-checks/` | самодостаточные HTML-мини-проверки знаний после инструктажа |
| `local-procedures/` | шаблоны по оборудованию, пропускам, ключам и парковке |
| `registers/` | спецификация и synthetic Excel-книга учета инструктажей |
| `interactive/` | интерактивный HTML-чек-лист выбора и отметки процедур |

## Быстрый старт

1. Заполнить `core/employee_input_form.md`.
2. Сформировать пакет по `core/onboarding_package_template.md`.
3. Подобрать доступы по `access/workgroup_access_matrix.md`.
4. Подобрать инструктажи по `instructions/primary_instruction_matrix_template.md`.
5. Проверить результат по `core/quality_checklist.md`.

## Public-safe правила

- Не копировать материалы из `doc/` и `instr/`.
- В примерах использовать только `TEST-*`, `example.test` и synthetic ФИО.
- Профильные работы, электроустановки и высоту помечать как `требует профильной проверки`.
- HTML-файлы должны быть self-contained: без CDN, внешних шрифтов, analytics и сетевых запросов.
- Excel-книгу создавать заново и держать только с synthetic строками.
