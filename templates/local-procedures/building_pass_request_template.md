# Шаблон заявки на электронный пропуск

> Public-safe template. Не содержит реальных маршрутов доступа, постов охраны и номеров пропусков.

## Заявка

| Поле | Значение |
|------|----------|
| Тип заявки | `[первичный пропуск / продление / временный доступ]` |
| Сотрудник | `[Примеров Петр Петрович]` |
| Роль | `[роль]` |
| Подразделение | `[TEST-DEPARTMENT]` |
| Объект | `[TEST-SITE]` |
| Зона доступа | `[TEST-ZONE]` |
| Срок действия | `[YYYY-MM-DD - YYYY-MM-DD]` |
| Основание | `[onboarding / заявка руководителя / договор]` |

## Согласование

| Роль | Статус |
|------|--------|
| Руководитель | `[согласовано / требуется]` |
| HR | `[согласовано / требуется]` |
| АХО / служба пропусков | `[оформлено / требуется]` |

## Предупреждение

Пропуск не должен оформляться автоматически по результату генерации агента; требуется проверка ответственными лицами.
