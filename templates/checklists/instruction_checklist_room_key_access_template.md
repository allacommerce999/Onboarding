# Чек-лист доступа к кабинету и ключам

> Template. Не указывать реальные кабинеты, ключницы, маршруты доступа и ФИО ответственных.

## Область применения

Для офисных сотрудников, которым требуется физический доступ к помещению или ключам.

## Ответственные

| Роль | Зона ответственности |
|------|----------------------|
| АХО | выдача ключа и учет |
| Руководитель | подтверждение необходимости |
| Сотрудник | соблюдение правил хранения |

## Простая проверка

- [ ] Руководитель подтвердил необходимость доступа.
- [ ] Указан synthetic кабинет: `[Кабинет TEST-101]`.
- [ ] Сотруднику объяснен порядок хранения и возврата ключей.
- [ ] Запись добавлена в список доступа или журнал выдачи.

## Фиксация

| Поле | Значение |
|------|----------|
| Кабинет | `[TEST-ROOM]` |
| Тип доступа | `[ключ / электронный доступ]` |
| Подтверждение | `[журнал / заявка / ЭДО]` |
