# Чек-лист оформления электронного пропуска

> Template. Не публиковать реальные номера пропусков и маршруты доступа.

## Область применения

Для сотрудников и подрядчиков, которым нужен доступ в здание или на объект.

## Ответственные

| Роль | Зона ответственности |
|------|----------------------|
| HR | проверка основания |
| АХО / служба пропусков | оформление пропуска |
| Руководитель | подтверждение зоны доступа |

## Простая проверка

- [ ] Заполнена заявка на пропуск с synthetic данными.
- [ ] Указана зона доступа без реального маршрута: `[TEST-ZONE]`.
- [ ] Срок действия пропуска ограничен или указан для пересмотра.
- [ ] Сотрудник получил правила использования пропуска.

## Фиксация

| Поле | Значение |
|------|----------|
| Тип пропуска | `[постоянный / временный / подрядчик]` |
| Зона | `[TEST-ZONE]` |
| Статус | `[требует согласования / оформлено]` |
