# Quickstart

Этот файл содержит полный сценарий запуска Onboarding Agent и единственную полную команду для чата.

## 1. Получить файлы

Скачайте репозиторий одним из способов:

- на GitHub: `Code -> Download ZIP`;
- через Git:

```powershell
git clone https://github.com/allacommerce999/Onboarding.git
```

Откройте папку `Onboarding`.

## 2. Заполнить форму

Откройте и заполните:

- `templates/core/employee_input_form.md`

Если часть данных неизвестна, оставьте `требует уточнения`.

## 3. Подготовить матрицу доступов

Используйте базовую матрицу:

- `templates/access/workgroup_access_matrix.md`

При необходимости адаптируйте роли, владельцев систем, сроки и правила согласования под свою организацию.

## 4. Открыть чат с агентом

Откройте чат, где будет использоваться Onboarding Agent, и вставьте команду ниже.

Если чат поддерживает вложения, можно приложить заполненную форму и матрицу доступов вместо вставки их полного текста.

```text
Подготовь onboarding-пакет в режиме draft.

Используй заполненную форму сотрудника и матрицу доступов ниже.
Если данных не хватает, отметь их как `требует уточнения` и вынеси вопросы в раздел `Открытые вопросы`.
Чувствительные доступы и профильные допуски не отмечай как выданные автоматически.

Форма:
[вставьте сюда заполненный файл templates/core/employee_input_form.md]

Матрица доступов:
[вставьте сюда файл templates/access/workgroup_access_matrix.md]
```

Для финальной версии замените в первой строке `draft` на `final`.

## 5. Проверить результат

После генерации проверьте пакет по чек-листу:

- `templates/core/quality_checklist.md`

Особенно проверьте:

- заполнены ли обязательные данные;
- есть ли у каждого доступа владелец, срок и статус;
- помечены ли чувствительные доступы как `требует согласования`;
- помечены ли профильные допуски как `требует профильной проверки`;
- содержит ли welcome-письмо дату выхода, формат работы и контактное лицо.
