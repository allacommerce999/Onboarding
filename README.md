# Onboarding Agent

Onboarding Agent - локальный генератор onboarding-пакетов для HR, PeopleOps, руководителя и наставника. Инструмент собирает папку нового сотрудника с отдельными документами: карточкой, планом адаптации, заявками, списками, инструктажами, проверками знаний и welcome-письмом.

Публичная версия использует только тестовые ресурсы и очищенные демонстрационные фрагменты. Доступы, пропуска, ключи, оборудование и инструктажи не считаются выданными без ручного подтверждения ответственных служб.

## Запуск

Из корня `Onboarding-Public`:

```powershell
.\tools\start_onboarding_agent.ps1
```

Откроется локальная страница `http://127.0.0.1:8765/`.

## Что создается

```text
employees/
└── YYYY-MM-DD_login/
    ├── 00_input/employee_input.json
    ├── 01_summary/
    ├── 02_access/
    ├── 03_local_procedures/
    ├── 04_instructions/
    ├── 05_email/
    └── Проверка.docx
```

Основной режим инструкций - `linked`: в пакет кладутся назначения и ссылки на канонические публичные фрагменты. Режим `self-contained` копирует выбранные инструкции и HTML-проверки в папку сотрудника.

## Карта

| Путь | Назначение |
|------|------------|
| `src/onboarding_app/` | локальный Python-сервер и генератор |
| `src/onboarding_app/web/` | HTML/CSS/JS интерфейс |
| `config/` | настройки домена, каталогов документов и инструкций |
| `templates/documents/` | очищенные публичные шаблоны `.docx` и `.xlsx` |
| `templates/knowledge-checks/` | HTML-проверки знаний |
| `examples/instructions/` | публичные адаптированные фрагменты инструкций |
| `employees/` | синтетические демо-пакеты и локальные результаты |
| `_legacy/2026-05-markdown-agent/` | архив старой markdown-версии, исключен из публикации |

## Ограничения

- реальные исходные документы и внутренние каталоги не копируются в публичную папку как есть;
- `.eml` создается как черновик и не отправляется автоматически;
- чувствительные доступы получают статус `требует согласования`;
- `_legacy/` не должен попадать в публичный архив или release package.
